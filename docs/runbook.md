# 운영 런북 (초안)

도메인 이벤트·관리자 화면 갱신 맥락은 [domain-events.md](./domain-events.md)를 함께 본다.

서비스가 느리거나, **5xx**, **결제·정산 오류**가 난다고 할 때 관리자·담당자가 **짧은 순서**로 확인하기 위한 문서입니다.  
코드 기준 관측: `frontend/src/lib/server/apiMonitoring.ts`, Sentry(`@sentry/nextjs`).

---

## 1. 바로 확인할 것 (1분)

1. **사용자에게 재현 정보**  
   - 시각(UTC+7 등), 브라우저, 로그인 여부(게스트/호스트/관리자), URL, 주문·예약 ID가 있으면 메모.
2. **배포·인프라 상태**  
   - 최근 배포가 있었는지, DB·Redis 등 외부 의존성 알림이 있는지.
3. **관리자 화면이면**  
   - 상단 배지·목록이 **오래된 값**인지: 다른 탭에서 조치 후 **새로고침 한 번**으로 정상화되는지 본다. (SSE 끊김 시 자동 재연결은 약 4초 지연 재시도.)

---

## 2. 서버 로그·액세스 로그

### JSON 한 줄 액세스 로그 (선택)

- 환경 변수: **`API_ACCESS_LOG=1`**
- 효과: 많은 API 응답이 완료될 때 **stdout에 JSON 한 줄**(`k: "api_route"`, `route`, `status`, `durationMs`, `at`)이 남는다.  
- 용도: 호스팅 플랫폼(Vercel, AWS, 자사 서버)의 **로그 수집기**에서 느린 라우트·4xx/5xx 비율을 빠르게 본다.
- 주의: 트래픽이 크면 로그량이 많아진다. **일시적으로 켜고** 원인 파악 후 끄는 것을 권장.

### 느린 요청 / 5xx → Sentry

- **`API_SLOW_MS`**: 밀리초. 기본은 코드상 **1200ms** 근처. 이보다 오래 걸리면 Sentry에 **`api_slow`**(경고)로 남을 수 있다.
- **HTTP 5xx**: Sentry에 **`api_http_5xx`**(에러)로 남는다.
- 라우트 핸들러에서 `observeRouteResponse`를 쓰는 경로는 응답에 **`Server-Timing: app;dur=...`** 헤더가 붙을 수 있다(브라우저 개발자 도구 → Network).

### 예외 처리

- 처리 중 잡히지 않은 예외는 `reportApiException` 등으로 Sentry **`captureException`**에 남기도록 연결된 구간이 있다.

**Sentry에서 할 일**

1. 프로젝트를 연다.
2. Issues에서 **최근 시간 필터** + 태그 **`route`**(또는 메시지 `api_http_5xx`, `api_slow`)로 검색.
3. 같은 `route`가 연쇄인지, 특정 `status`·DB 타임아웃인지 본다.

**알림·담당·임계값 상세:** [docs/qa/sentry-ops.md](./qa/sentry-ops.md) (Phase 2-3). 팀은 해당 문서의 알림 규칙·온콜 표를 실제 이름으로 채운다.

**상용 진입 전 전역 클로저:** [docs/qa/pre-launch-closure.md](./qa/pre-launch-closure.md) (법무·연동 일정·웹훅·보안·다국어·게이트).

## 3. 결제·정산 관련일 때

1. **해당 예약·결제 ID**로 DB에서 `Booking`, `PaymentRecord`, 정산/환불 관련 테이블 상태를 확인한다. (스키마는 Prisma `frontend/prisma/schema.prisma`.)
2. **관리자 API**  
   - 정산·환불·계약·출금 화면은 도메인 이벤트와 연동되어 있음 → `docs/domain-events.md` 참고.
3. **PG·외부 결제 대시보드**가 있다면, 동일 시각의 **승인/실패/웹훅** 로그와 대조한다.
4. 클라이언트만 오류일 때: 네트워크 탭에서 실패한 **`/api/...`**, 응답 본문, `Server-Timing`을 캡처한다.

---

## 4. 데이터베이스·환경 변수

1. **`DATABASE_URL`**  
   - 앱 기동 시 잘못된 따옴표·공백 등을 정규화하는 코드가 있다. 복사-붙여넣기 오류가 있으면 연결 실패·간헐 오류가 난다.
2. **Sentry DSN** 등 비밀 값은 저장소에 커밋하지 말고, 호스팅 콘솔의 **환경 변수**로만 설정한다.
3. 설정을 바꾼 뒤에는 **재배포 또는 프로세스 재시작**이 필요한지 플랫폼 규칙을 확인한다.

---

## 5. 장애 등급별 권장 대응

| 등급 | 예시 | 권장 |
|------|------|------|
| 전체 다운 | 모든 요청 5xx | `API_ACCESS_LOG` 일시 ON → 원인 라우트 확인 → 롤백 또는 핫픽스 |
| 부분 기능 | 결제만 실패 | Sentry `route` 필터 + PG 로그 + 해당 `POST/GET` 재현 |
| 느림 | 타임아웃 다발 | `api_slow` 분포, DB 슬로우 쿼리, `API_SLOW_MS` 조정은 **원인 확인 후** |

---

## 6. 이 런북을 업데이트할 때

- 새로 **관측을 붙인 API**가 있으면 “어떤 태그/메시지로 Sentry에 남는지” 한 줄씩 추가한다.
- 실제 운영 팀이 쓰는 **호스팅 콘솔 이름**(예: CloudWatch, Vercel Logs)을 절에 링크나 스크린샷 절차로 보강한다.
