# Phase 4 — 스테이징 점검 체크리스트

배포 전 기능·보안·다국어·거래 흐름을 한 번에 확인할 때 사용합니다.

## 이메일 템플릿 (다국어·레이아웃)

트랜잭션·보안·KYC 관련 메일은 **언어별로 별도 HTML**을 두거나, 단일 템플릿에 **locale 파라미터**로 분기합니다.

### 언어·방향

- **LTR**: 한국어(ko), 베트남어(vi), 영어(en), 일본어(ja), 중국어 간체(zh-CN)는 기본적으로 왼쪽에서 오른쪽입니다. 루트 요소에 `dir="ltr"`를 명시해 두면 일부 클라이언트에서 혼선을 줄일 수 있습니다.
- **RTL**: 아랍어·히브리어 등을 추가할 경우에만 `dir="rtl"`과 `text-align`·아이콘·플로트를 거울로 맞춥니다. 현재 제품 5개국어는 LTR만 사용합니다.
- **문자 인코딩**: UTF-8, `<meta charset="utf-8">` 고정.

### 치환자(플레이스홀더) 규칙

- 형식은 **`{{snake_case}}`** 로 통일합니다. 예: `{{user_email}}`, `{{booking_id}}`, `{{amount_vnd}}`, `{{support_email}}`, `{{operator_legal_name}}`.
- **HTML 이스케이프**: 사용자 입력·주소 등이 들어가는 값은 서버에서 이스케이프한 뒤 삽입합니다. 신뢰된 운영자 고정 문구만 `v-html` 급으로 넣지 않습니다.
- **날짜·금액**: 가능하면 서버에서 **이미 포맷된 문자열**을 넘깁니다(예: `{{check_in_date_localized}}`). 그렇지 않으면 locale + 타임존을 쿼리 파라미터나 헤더와 함께 명시합니다.
- **조건부 블록**: `{{#if optional_section}}...{{/if}}` 같은 최소 문법을 쓰거나, locale별 partial 템플릿으로 나눕니다. 빈 문자열이 나가지 않도록 기본 문장을 둡니다.
- **미번역 폴백**: 특정 언어 템플릿이 없으면 **영문(en)** 본문으로 보내고, 푸터에 “English version” 안내를 붙이는 방식을 권장합니다.

### QA

- 각 언어별로 **제목(Subject)** 과 **본문 첫 문단**이 자연스러운지, 깨진 치환자 `{{...}}` 가 남아 있지 않은지 확인합니다.
- 모바일 메일 클라이언트에서 **단일 컬러**·**최대 너비(예: 600px)** 로 깨짐이 없는지 스팟 체크합니다.

## 앱·관리자 스모크 (요약)

- 로그인·가입·검색·예약·결제·KYC·정산·출금·약관/개인정보·관리자 정산/회원 차단까지 주요 경로를 **KO/VI/EN/JA/ZH** 각각에서 한 번씩 확인합니다.
- 자동 점검: `npm run check:banned-terms`, `npm test`(UI 키 패리티·App API 오류 사전 패리티 포함).
